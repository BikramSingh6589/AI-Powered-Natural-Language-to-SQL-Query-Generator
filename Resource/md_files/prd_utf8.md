<p><strong>Software Requirements Specification (SRS)</strong></p><p><strong>AI-Powered Natural Language to SQL Query Generator</strong></p><p>Version: 1.0</p><p><strong>1. Introduction</strong></p><p><strong>1.1 Purpose</strong></p><p>The purpose of this project is to develop an intelligent AI-powered system that converts natural language user requirements into SQL queries. The system enables users with little or no database knowledge to interact with databases using simple English sentences and automatically generate valid SQL statements.</p><p>The application reduces the complexity of writing SQL queries and assists users in understanding database operations through AI-generated explanations and query recommendations.</p><p><strong>1.2 Scope</strong></p><p>The Natural Language to SQL Query Generator is designed to:</p><ul><li>Accept user requirements in natural language.</li><li>Analyze user intent using Artificial Intelligence and Natural Language Processing (NLP).</li><li>Generate one or more SQL queries corresponding to the user's request.</li><li>Explain generated SQL queries in simple language.</li><li>Display tables, attributes, and relationships used in the query.</li><li>Validate generated queries before execution.</li><li>Execute queries on connected databases.</li><li>Display query results in tabular format.</li><li>Help non-technical users interact with databases without learning SQL syntax.</li></ul><p>The system is intended for students, analysts, researchers, business users, and database beginners.</p><p><strong>1.3 Definitions, Acronyms, and Overall Description</strong></p><p><strong>2.1 Product Perspective</strong></p><p>The system acts as an intelligent layer between users and databases.</p><p>Instead of writing SQL manually, users describe their requirements in plain English. The AI engine interprets the request and generates appropriate SQL statements.</p><p>Example:</p><p>User Input:</p><p>"Show all students whose marks are greater than 80."</p><p>Generated SQL:</p><p>SELECT * FROM students</p><p>WHERE marks &gt; 80;</p><p><strong>2.2 Product Functions</strong></p><p>The system shall:</p><ol><li>Accept natural language input.</li><li>Analyze user intent.</li><li>Identify database entities and attributes.</li><li>Generate SQL queries.</li><li>Provide multiple query alternatives when applicable.</li><li>Explain generated queries.</li><li>Execute generated queries.</li><li>Display results.</li><li>Save query history.</li><li>Provide schema visualization.</li><li>Validate SQL syntax.</li><li>Suggest query improvements.</li></ol><p><strong>2.3 User Classes and Characteristics</strong></p><p><strong>Database Beginner</strong></p><ul><li>No SQL knowledge</li><li>Uses natural language only</li></ul><p><strong>Student</strong></p><ul><li>Learning SQL</li><li>Uses explanations to understand queries</li></ul><p><strong>Data Analyst</strong></p><ul><li>Uses generated SQL for faster reporting</li></ul><p><strong>Database Administrator</strong></p><ul><li>Verifies generated SQL before execution</li></ul><p><strong>2.4 Operating Environment</strong></p><p><strong>Client Side</strong></p><ul><li>Web Browser</li><li>Chrome</li><li>Firefox</li><li>Edge</li></ul><p><strong>Server Side</strong></p><ul><li>Python</li><li>Django Framework</li><li>PostgreSQL/MySQL</li></ul><p><strong>AI Layer</strong></p><ul><li>OpenAI API</li><li>NLP Processing Engine</li></ul><p><strong>2.5 Assumptions and Dependencies</strong></p><p><strong>Assumptions</strong></p><ul><li>Database schema is available.</li><li>Internet connection is available for AI processing.</li><li>Users provide meaningful natural language inputs.</li></ul><p><strong>Dependencies</strong></p><ul><li>Database Server</li><li>AI Service API</li><li>Django Backend</li><li>Frontend Framework</li></ul><p><strong>2.6 Constraints</strong></p><ul><li>AI-generated queries may require validation.</li><li>Query accuracy depends on schema availability.</li><li>Internet dependency for AI features.</li></ul><p><strong>3. Specific Requirements</strong></p><p><strong>3.1 Functional Requirements</strong></p><p><strong>FR-1 User Authentication</strong></p><p><strong>Description</strong></p><p>The system shall allow users to register and log in.</p><p><strong>Inputs</strong></p><ul><li>Username</li><li>Email</li><li>Password</li></ul><p><strong>Outputs</strong></p><ul><li>Successful login</li><li>Error message</li></ul><p><strong>FR-2 Natural Language Input</strong></p><p><strong>Description</strong></p><p>The system shall allow users to enter database requests in natural language.</p><p><strong>Example</strong></p><p>"Show employees working in the HR department."</p><p><strong>FR-3 Intent Recognition</strong></p><p><strong>Description</strong></p><p>The system shall identify:</p><ul><li>Tables</li><li>Attributes</li><li>Conditions</li><li>Aggregations</li></ul><p>from the user input.</p><p><strong>FR-4 SQL Query Generation</strong></p><p><strong>Description</strong></p><p>The system shall generate valid SQL queries from natural language inputs.</p><p><strong>Example</strong></p><p>Input:</p><p>"Find students older than 20."</p><p>Output:</p><p>SELECT *</p><p>FROM students</p><p>WHERE age &gt; 20;</p><p><strong>FR-5 Multiple Query Suggestions</strong></p><p><strong>Description</strong></p><p>The system shall generate alternative SQL queries when multiple interpretations exist.</p><p><strong>FR-6 Query Explanation</strong></p><p><strong>Description</strong></p><p>The system shall explain generated SQL queries in simple language.</p><p>Example:</p><p>"This query retrieves all students whose age is greater than 20."</p><p><strong>FR-7 Schema Detection</strong></p><p><strong>Description</strong></p><p>The system shall identify database tables and attributes relevant to the user query.</p><p><strong>FR-8 Query Validation</strong></p><p><strong>Description</strong></p><p>The system shall validate SQL syntax before execution.</p><p><strong>FR-9 Query Execution</strong></p><p><strong>Description</strong></p><p>The system shall execute validated SQL queries on the connected database.</p><p><strong>FR-10 Result Visualization</strong></p><p><strong>Description</strong></p><p>The system shall display query results in tabular format.</p><p><strong>FR-11 Query History</strong></p><p><strong>Description</strong></p><p>The system shall store previously generated queries.</p><p><strong>FR-12 Export Functionality</strong></p><p><strong>Description</strong></p><p>The system shall allow exporting results as:</p><ul><li>CSV</li><li>Excel</li><li>PDF</li></ul><p><strong>3.2 Non-Functional Requirements</strong></p><p><strong>NFR-1 Performance</strong></p><ul><li>Query generation time shall be less than 5 seconds.</li><li>Result retrieval shall be completed within 10 seconds for normal datasets.</li></ul><p><strong>NFR-2 Security</strong></p><ul><li>Passwords shall be encrypted.</li><li>Database credentials shall not be exposed.</li><li>SQL injection protection shall be implemented.</li><li>HTTPS shall be used.</li></ul><p><strong>NFR-3 Reliability</strong></p><ul><li>System uptime shall exceed 95%.</li><li>Failed query executions shall generate meaningful error messages.</li></ul><p><strong>NFR-4 Usability</strong></p><ul><li>Interface shall be simple and beginner-friendly.</li><li>Users shall be able to generate queries without SQL knowledge.</li></ul><p><strong>NFR-5 Scalability</strong></p><ul><li>System shall support multiple concurrent users.</li><li>System shall support large databases.</li></ul><p><strong>NFR-6 Maintainability</strong></p><ul><li>Modular architecture shall be followed.</li><li>APIs shall be documented.</li></ul><p><strong>NFR-7 Compatibility</strong></p><p>The system shall support:</p><ul><li>Windows</li><li>Linux</li><li>macOS</li></ul><p>and modern browsers.</p><p><strong>4. External Interface Requirements</strong></p><p><strong>4.1 User Interface</strong></p><p>The interface shall include:</p><ul><li>Login Page</li><li>Dashboard</li><li>Query Input Box</li><li>Generated SQL Section</li><li>Query Explanation Section</li><li>Results Table</li><li>Query History Page</li></ul><p><strong>4.2 Hardware Interfaces</strong></p><p>Minimum Requirements:</p><ul><li>4 GB RAM</li><li>Dual Core Processor</li><li>Internet Connectivity</li></ul><p><strong>4.3 Software Interfaces</strong></p><p><strong>Database</strong></p><ul><li>PostgreSQL</li><li>MySQL</li></ul><p><strong>AI Services</strong></p><ul><li>OpenAI API</li></ul><p><strong>Backend</strong></p><ul><li>Django REST Framework</li></ul><p><strong>5. System Features</strong></p><p><strong>Feature 1: Natural Language Processing</strong></p><p>Convert English text into structured query requirements.</p><p><strong>Feature 2: SQL Generation Engine</strong></p><p>Generate optimized SQL statements.</p><p><strong>Feature 3: Query Explanation Engine</strong></p><p>Explain SQL queries in human-readable language.</p><p><strong>Feature 4: Query Execution Module</strong></p><p>Execute SQL against connected databases.</p><p><strong>Feature 5: Result Visualization</strong></p><p>Display results in structured tables.</p><p><strong>6. Security Requirements</strong></p><ul><li>Authentication required.</li><li>Role-based access control.</li><li>Encrypted passwords.</li><li>Secure database connection.</li><li>Input validation.</li><li>SQL Injection prevention.</li></ul><p><strong>7. Future Enhancements</strong></p><ul><li>Voice-to-SQL generation.</li><li>Multi-language support.</li><li>NoSQL query generation.</li><li>Query optimization recommendations.</li><li>Database schema auto-discovery.</li><li>AI chatbot assistant.</li></ul><p><strong>8. Acceptance Criteria</strong></p><p>The system shall be considered successful if:</p><ol><li>Natural language inputs are converted into valid SQL queries.</li><li>Generated SQL executes successfully.</li><li>Query explanations are understandable by beginners.</li><li>Results are displayed correctly.</li><li>Users can interact without SQL knowledge.</li></ol><p><strong>9. Conclusion</strong></p><p>The AI-Powered Natural Language to SQL Query Generator aims to bridge the gap between non-technical users and database systems. By leveraging Artificial Intelligence and Natural Language Processing, the system enables users to generate, understand, and execute SQL queries through natural language, improving accessibility, productivity, and learning.</p>
